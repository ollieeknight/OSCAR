    Getting Started

    OSCAR is a Nextflow pipeline. A single command takes you from BCL output to per-library QC. Containers are pulled automatically via Apptainer; no manual image management is needed.

    Prerequisites

    • BIH cluster access (or any Linux system with SLURM)
    • Nextflow ≥24.x (module load nextflow, or install to ~/bin)
    • Apptainer/Singularity

    Installation

    git clone https://github.com/ollieeknight/OSCAR ~/bin/OSCAR

    Reference genome paths are pre-configured in nextflow.config to point at the shared project storage on BIH. No additional setup is needed if you are on the cluster.

    Prepare Metadata

    Create a metadata.csv describing your libraries, or use the Metadata Generator tool.

    Required columns:

    assay,experiment_id,historical_number,replicate,modality,chemistry,index_type,index,species,n_donors,adt_file
CITE,EXP001,H001,R1,GEX,3prime,dual,SI-TT-A1,human,1,adt_totalseq_b.csv

    ADT CSV files are resolved relative to the samplesheet directory at adt_files/{adt_file}.csv by default. Use --adt_files_dir to override this.

    Running the Pipeline

    From BCL (recommended)

    Full run from raw sequencer output:

    nextflow run ~/bin/OSCAR/main.nf     --samplesheet metadata.csv     --bcl_dir /path/to/BCL     --outdir results     --run_name R001     -profile slurm

    From FASTQ

    Skip demultiplexing if FASTQs already exist:

    nextflow run ~/bin/OSCAR/main.nf     --samplesheet metadata.csv     --from_fastq     --fastq_dir /path/to/fastq     --outdir results     -profile slurm

    From Cell Ranger outputs

    Start from existing outs/ directories to run QC only:

    nextflow run ~/bin/OSCAR/main.nf     --samplesheet metadata.csv     --from_cellranger     --outs_dir /path/to/outs     --outdir results     -profile slurm

    Multiple BCL runs

    For experiments spread across multiple flowcells, pass the additional runs as comma-separated lists:

    nextflow run ~/bin/OSCAR/main.nf     --samplesheet metadata_R001.csv     --bcl_dir /path/to/R001     --extra_bcl_dirs /path/to/R002,/path/to/R003     --extra_samplesheets metadata_R002.csv,metadata_R003.csv     --outdir results     -profile slurm

    Configuration

    All parameters live in nextflow.config. The most commonly overridden ones at runtime:

    ParameterDefaultDescription
    --outdirresultsOutput directory
    --run_namerunPrefix for FASTQ/outs subdirectory names
    --run_untilnullStop after FASTQ or cellranger
    --sequencernovaseq_xFallback when no BCL directory is provided; ignored otherwise

    Reference genome and container paths are set in nextflow.config and pre-configured for the BIH cluster. Edit those lines directly if you need to point at a different location.

    Troubleshooting

    Resuming a failed run

    Add -resume to any nextflow run command to restart from the last successful step without rerunning completed processes.

    BCL Convert demultiplexing errors

    Check that every library in metadata.csv has the correct index value and index_type (dual or single). Dual-index libraries require a 10 bp i7 + 10 bp i5 SI-TT or SI-TN index.

    Memory errors on large runs

    Resource limits are per-process in nextflow.config under the slurm profile. Increase memory or time for the relevant label (process_high, CELLRANGER_MULTI, etc.).

    Checking logs

    Nextflow writes a .nextflow.log in the launch directory and per-process logs under work/<hash>/. Run nextflow log to list past runs and locate their work directories.
