Metadata file generator

Creates the metadata.csv file needed to run OSCAR. Fill in one row per library per sequencing run, then download.

Field guide — what goes in each column
Assay type
The sequencing assay: CITE (RNA + antibody), DOGMA (RNA + ATAC + antibody), ASAP (ATAC + antibody), Multiome (RNA + ATAC), ATAC, or GEX (RNA only)
Experiment ID
A short label used in file and directory names — no spaces (e.g. JIA_exp1, CD34_stim)
Historical number
Cell pool number. Cells loaded from the same pool into more than one GEM well all share one historical number (e.g. pool 1 → wells A and B both get historical number 1)
Replicate
Distinguishes GEM wells from the same pool: A for the first well, B for the second, and so on
Modality
The data type for this row: gene expression (GEX), surface antibody (ADT), hashtag (HTO), chromatin accessibility (ATAC), T or B cell receptor (VDJ-T / VDJ-B)
Chemistry
The 10x Genomics kit chemistry. Check the kit box or your sequencing order — if unsure, ask whoever prepared the library
Index type
Single or dual index. Dual-index kits (SI-TT, SI-TN) use two barcodes; single-index kits (SI-GA) use one. Check your index plate lid
Index
The index well ID from your plate (e.g. SI-TT-A1). Found on the plate lid or your sequencing request form
Species
Human or mouse
Number of donors
How many individuals are pooled in this GEM well. Write 1 if not multiplexed. Used for donor deconvolution (human only)
ADT CSV name
Filename of the antibody barcode reference, without the .csv extension. Leave blank if this row has no antibody capture
